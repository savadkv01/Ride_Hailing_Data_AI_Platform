# Ride-Hailing Data & AI Platform - Stage Index

This file is the canonical reference for the staged execution flow.

## Stage 0 – Business Context & Ride-Hailing Domain Modeling
Define business KPIs, ride lifecycle, actors, and domain boundaries.

## Stage 1 – Enterprise Ride Data Architecture
Design end-to-end architecture across operational systems, data platform, and AI layers.

## Stage 2 – Deep Theory Foundation
Cover streaming architecture, lakehouse vs warehouse, real-time ETL, dimensional modeling, feature engineering, vector databases, RAG, demand forecasting, dynamic pricing, and fraud detection.

## Stage 3 – Data Source Strategy (Open + Synthetic)
Select open datasets and design synthetic enterprise-grade ride-hailing data.

## Stage 4 – AI-Ready Data Modeling
Design operational, dimensional, and vector-ready data models.

## Stage 5 – Full Docker Infrastructure Setup
Set up the full local stack on Windows using Docker and Docker Compose.

## Stage 6 – Kafka Ingestion & Event Simulation
Implement topic design, event contracts, producers, and simulation streams.

## Stage 7 – Spark Structured Streaming (Bronze/Silver/Gold)
Build streaming pipelines from raw to curated layers.

## Stage 8 – Dimensional Modeling with dbt (PostgreSQL Warehouse)
Create dbt models, tests, and documentation for analytical star schemas.

## Stage 9 – ML Modeling & Feature Pipeline
Build feature pipelines and models for demand, surge, churn, and fraud.

## Stage 10 – Vector Embedding Pipeline
Generate embeddings and store them in the selected open-source vector database.

## Stage 11 – RAG-based Ride Intelligence Assistant
Implement retrieval-augmented intelligence for support, analytics, and investigations.

## Stage 12 – FastAPI Platform API Layer
Expose APIs for analytics, model inference, and RAG services.

## Stage 13 – Observability & Logging
Implement metrics, structured logging, data quality monitoring, and alerts.

## Stage 14 – Enterprise Scalability & Multi-City Expansion
Design for high throughput, partitioning strategy, and multi-city scaling.

## Stage 15 – Security, Governance & Data Contracts
Implement governance, contracts, access controls, lineage, and compliance patterns.

## Stage 16 – Azure Enterprise Migration Blueprint
Map open-source components to Azure-native services and migration strategy.

## Stage 17 – CI/CD & Production Hardening
Implement testing, release automation, resilience, and operational runbooks.
